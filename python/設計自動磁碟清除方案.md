## 實現方法
要實現“若使用者超過 30 天未開機，則自動清洗整個硬碟”這個功能，大致上需要三個步驟：

1. **背景服務/自動啟動程式**  
    在每次電腦開機時，就要有一個程式（或服務）自動執行。
    
    - Windows：寫成 Windows Service，或簡單地用 Task Scheduler 排程「開機時執行」。
        
    - Linux：寫成 systemd service 或在 `/etc/rc.local`／cron @reboot。
        
2. **紀錄與檢查上次執行時間**  
    每次程式啟動時，先讀取上次執行時留下的時間戳（存在 Registry、檔案或資料庫），計算與當前時間的差距：
    
    ```text
    if (current_time – last_run_time) ≥ 30 天 → 觸發清洗流程
    else → 更新 last_run_time 為 current_time，結束程式
    ```
    
3. **呼叫系統指令或原始磁碟存取進行「清洗」**  
    一般做法是透過格式化／覆寫原始磁區（需以管理員／root 權限執行）。
    
    - Windows：可以用 `diskpart`、WMI、或 Sysinternals 的 `sdelete`。
        
    - Linux：可以用 `dd if=/dev/zero of=/dev/sda bs=1M`、`wipe`，但注意：無法對正在掛載的根分區做完整覆寫，通常要從 Live USB/PE 環境執行，或先卸載目標磁區。

## 一、Windows 平台範例（Python + Task Scheduler）

1. **Python 腳本 `auto_wipe.py`**
    
    ```python
    import os, sys, subprocess
    from datetime import datetime, timedelta
    import winreg
    
    # Registry 路徑與鍵值
    KEY_PATH   = r"SOFTWARE\MyAutoWipe"
    VALUE_NAME = "LastRunISO"
    
    def get_last_run():
        try:
            key = winreg.OpenKey(winreg.HKEY_LOCAL_MACHINE, KEY_PATH)
            val, _ = winreg.QueryValueEx(key, VALUE_NAME)
            winreg.CloseKey(key)
            return datetime.fromisoformat(val)
        except:
            return None
    
    def set_last_run(dt):
        key = winreg.CreateKey(winreg.HKEY_LOCAL_MACHINE, KEY_PATH)
        winreg.SetValueEx(key, VALUE_NAME, 0, winreg.REG_SZ, dt.isoformat())
        winreg.CloseKey(key)
    
    def is_admin():
        # 簡單檢查是否以管理員身份執行
        try:
            return os.getuid() == 0
        except:
            import ctypes
            return ctypes.windll.shell32.IsUserAnAdmin() != 0
    
    def wipe_disk():
        # 範例：用 diskpart 來格式化 C:（請小心！）
        script = "\n".join([
            "select volume C",
            "format fs=NTFS quick label=ERASED",
            "exit"
        ])
        p = subprocess.Popen(['diskpart'], stdin=subprocess.PIPE, text=True)
        p.communicate(script)
    
    def main():
        if not is_admin():
            print("請以管理員身分執行")
            sys.exit(1)
    
        now  = datetime.now()
        last = get_last_run()
        if last and (now - last) >= timedelta(days=30):
            wipe_disk()
        else:
            set_last_run(now)
    
    if __name__ == "__main__":
        main()
    ```
    
2. **排程設定**
    
    - 開啟「排程工作程式」→ 建立基本工作 → 觸發：『At startup』→ 動作：執行 `python C:\path\auto_wipe.py`，勾選『使用最高權限執行』。

## 二、Linux 平台範例（Bash + systemd）

1. **Shell 腳本 `/usr/local/bin/auto_wipe.sh`**
    
    ```bash
    #!/bin/bash
    LASTFILE="/etc/auto_wipe_last"
    now=$(date +%s)
    
    # 第一次執行：只記錄時間
    if [ ! -f "$LASTFILE" ]; then
        echo $now > "$LASTFILE"
        exit 0
    fi
    
    last=$(cat "$LASTFILE")
    days=$(( (now - last) / 86400 ))
    
    if [ $days -ge 30 ]; then
        # 注意：如果要覆寫根分區，必須先 boot 到 Live/CD 環境
        dd if=/dev/zero of=/dev/sda bs=1M status=progress
    else
        echo $now > "$LASTFILE"
    fi
    ```
    
    ```bash
    chmod +x /usr/local/bin/auto_wipe.sh
    ```
    
2. **systemd 服務檔 `/etc/systemd/system/auto_wipe.service`**
    
    ```ini
    [Unit]
    Description=Auto wipe disk after 30 days inactivity
    After=network.target
    
    [Service]
    Type=oneshot
    ExecStart=/usr/local/bin/auto_wipe.sh
    
    [Install]
    WantedBy=multi-user.target
    ```
    
    ```bash
    systemctl daemon-reload
    systemctl enable auto_wipe.service
    ```
    

每次開機時，systemd 都會執行這個 service，依邏輯決定是否要執行覆寫整顆硬碟。

### 注意事項

1. **破壞性極高**：一旦執行，所有資料無法復原。務必在測試環境反覆驗證。
    
2. **掛載中分區**：大多數作業系統無法在正常執行時格式化自己所在的根分區，必須在外部環境（Live USB/WinPE）下執行。
    
3. **時鐘操控**：使用者可透過修改系統時間來繞過判斷。如需更嚴謹，可結合網路時間伺服器 (NTP) 驗證。
    
4. **權限**：一定要以 Administrator（Windows）或 root（Linux）身份執行，才能存取原始磁碟。
    

以上就是實作思路＋範例程式。你可以根據熟悉的程式語言（C#, Go, Rust……）改寫同樣的邏輯，並用對應平臺的服務/排程機制來部署。
