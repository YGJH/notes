### 1. 詞法分析 (Lexical Analyzer / Scanning)

這是編譯的第一步。編譯器讀入原始程式碼的字元流，並將其轉換為具備語法意義的**標記（Tokens）**。

- **功能：** 去除空白、註解，並辨識出關鍵字、識別碼（變數名）、常數及運算子。
    
- **範例：** 將 `position = initial + rate * 60` 拆解為 `id1`、`=`、`id2`、`+`、`id3`、`*`、`60`。
    

### 2. 語法分析 (Syntax Analyzer / Parsing)

此階段將 Tokens 排列成一個階層式結構，通常稱為**語法樹（Syntax Tree）**。

- **功能：** 檢查程式碼是否符合程式語言的語法規則（例如：括號是否成對、敘述句結尾是否正確）。
    

### 3. 語意分析 (Semantic Analyzer)

這一步是檢查程式碼的「邏輯意義」是否正確。

- **功能：** 進行**型別檢查（Type Checking）**。例如：確保你沒有把一個字串加到整數上，或者確保變數在使用前已經宣告。
    

### 4. 中間碼生成 (Intermediate Code Generator)

編譯器會產生一種介於原始碼與機器碼之間的低階表示法（如三位址碼 Three-Address Code）。

- **功能：** 這種形式易於處理且不依賴特定的硬體架構，方便後續進行優化。
    

### 5. 代碼優化 (Code Optimizer)

對中間碼進行調整，目的是讓程式運行得**更快**或**佔用更少記憶體**。

- **功能：** 刪除無效程式碼（Dead Code）、簡化數學運算（如 $x = x + 0$）或進行迴圈優化。
    

### 6. 代碼生成 (Code Generator)

最後一步，將優化後的中間碼轉換為目標機器的**機械碼（Machine Code）**或組合語言。

- **功能：** 分配暫存器並生成實際的指令流。
    

---

### 核心支持模組 (橫跨所有階段)

- **符號表管理器 (Symbol-Table Manager)：**
    
    這是一個資料結構，用來記錄程式中出現的所有**識別碼（變數名、函數名）**及其屬性（型別、作用域、存儲位置）。編譯器的每一個階段都會與它互動。
    
- **錯誤處理器 (Error Handler)：**
    
    當編譯器在任何階段發現錯誤（語法錯誤、型別不合等）時，錯誤處理器會負責收集這些資訊，並回報給使用者精確的錯誤訊息與位置。
