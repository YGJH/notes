# 簡介

- 此論文在討論如何線上分配分的好
	- **零羨慕 (Envy-freeness)**
	- **策略穩健性** : 策略在不同情況下都能保持有效性，並且能夠抵抗各種風險和不確定性的能力
	- **帕累托效率（Pareto efficiency），又稱帕累托最適** : 指的是在資源配置的情況下，不可能再增加一個人的福利，也不可能在減少一個人的福利下，現在這個情況就是最好的，稱作現在這個情況為帕雷托效率



## **名詞解釋**

### #抗操控（說真話划算嗎？）

- **SP — Strategy-Proof**  
全程看完所有到貨順序與機制時，**說真話永遠不吃虧**；任何整體的「假裝」出價都不會讓你期望效用更好。
- **OSP — Online Strategy-Proof**  
    **線上弱化版**：把過去決定視為固定，只看**當下這一件**到貨時，說真話也不會更差（對每一回合都成立）。SP ⇒ OSP，但 OSP 未必 ⇒ SP。

### #公平（會不會眼紅？）

- **EFA — Envy-Free ex Ante**  
**抽籤前、看期望**不嫉妒：在機率分配（lottery）層次，沒有人覺得別人的期望比自己好。表中的 EFA 就是這個意思；像 LIKE 機制是「在期望上無嫉妒」。
- **SEFA — Shared Envy-Free ex Ante**  
**更強的 EFA**：只比較兩人**都喜歡**的那些物品（不喜歡的拿去也不會讓我嫉妒），在期望層次也不嫉妒；SEFA ⇒ EFA
- **EFP — Envy-Free ex Post**  
    **抽籤後、實現的分配**就不嫉妒（很硬！）——每個實際結果都要無嫉妒。
- **SEFP — Shared Envy-Free ex Post**  
**EFP 的 shared 版本**：只在兩人都喜歡的物品上，實現後也不嫉妒。
- **BEFP — Bounded Envy-Free ex Post**  
**實現後的嫉妒有上界**（表中名詞）：機制保證任何人的嫉妒都不會「無限長大」；和文獻裡談的「嫉妒隨到貨數量漸消（vanishing envy）」是不同的控制方式。

### #效率（還能不能一起變更好？）

- **PEA — Pareto Efficient ex Ante**  
    在**機率分配**層次無法再做一個 lottery，讓所有人期望都不差且至少一人更好。
    
- **PEP — Pareto Efficient ex Post**  
    **每個實現的分配**都帕累托有效（沒有別的實現能讓至少一人更好且他人不更差）。


### #關係

- **SP ⇒ OSP**；反之不成立。
    
- **SEFA/SEFP ⇒ EFA/EFP**；反之不成立（shared 是更強的口徑）。
    
- **一般效用下**：沒有線上機制能同時 **EFA + PEA**（兩者一起不可能）。
    
- **EFA + PEP** 的組合也屬於圖中標記為「∅」的**不可能區**。





## 離線跟線上分配

- 簡單來說:
	- 線上就是大家都不知道未來會有甚麼物品
	- 離線就是大家有多少人、物品有哪些都已經知道了

## 內容

- 當場景是離線的:
	- 就是以代理人為視角(agent-centered) 其中可以一個或多個代理人從一組物品選擇

- 但當場景是線上:
	- 就會有物品為視角(item-centric) 因為是依照時間抵達順序出現的。


- **Like 機制**
	- 當物品抵達時，會分配給已經對該物品宣告positive reward的代理人
	- 此方法是符合**零羨慕** 、**策略穩健** 
	- 並且可以被視為**離線機率序列機制**
	- 並對於 0/1 效用，LIKE機制可以同時實現 零羨慕 跟 帕雷托效率 ( Pareto efficient ex ante )

**ONLINE SERIAL DICTATOR**
- **online serial dictator** 線上獨裁者:
	- 在第一輪之前，代理人有一個嚴格的優先順序，對於接下來的物品，唯一可行的是代理人中*第一個* 對該物品出價為正的代理人
- **balance like** 機制:
	- 在分配的時候，在很多代理人提出 positive value 時，還是要根據現在哪個agent分配到的物品最少而給最少的
	- 但是在 0/1 效用下，balance like 並不是策略穩健的
- **PARETO LIKE**
	- 只有當將物品分配給某個代理人會達成事後帕累托效率 (Pareto efficient ex post) 時，該代理人才被視為該物品的可行選擇

 - **MAXIMUM LIKE (最大類)**：一個代理人只有在對物品出價最高時，才被視為該物品的可行選擇。

-  **開發物品為中心的機制**：許多不可分割物品的線上公平分配機制，可以描述為在對抵達物品「可行」的代理人子集之間均勻選擇。這些機制包括：

   - **ONLINE RANDOM PRIORITY (線上隨機優先)**：在第一輪前隨機抽取代理人的嚴格優先順序，然後運行 ONLINE SERIAL DICTATOR。


   - **LIKE (類)**：代理人只要對物品宣告正效用即可被視為可行。

   - **MAXIMUM LIKE (最大類)**：將物品分配給對其出價最高的代理人。



## 2. 考慮放寬的規範屬性

由於線上環境的即時性和不確定性，實現某些強烈的公平屬性可能非常困難甚至不可能。因此，可以考慮採用較弱但更容易實現的規範屬性：

• **線上策略穩健性 (Online Strategy-Proofness, OSP)**：這是一種較弱的策略穩健性定義，它假設過去的決策已經固定，代理人只能對當前的決策採取策略。由於未來的不確定性減少了代理人的戰略選擇，因此在線上設定中實現線上策略穩健性比實現完全的策略穩健性更容易。

• **共享零羨慕 (Shared Envy-Freeness)**：這要求每對代理人只對彼此共同喜歡的物品感到零羨慕。如果我得到一個你不喜歡的物品，這不會引起你任何羨慕。

## 3. 理解不可能性結果

認識到在線上設定中無法同時實現的規範屬性組合，有助於避免追求不可能的目標，並指導機制設計。

• 對於一般（非 0/1）效用，**沒有任何線上機制可以同時實現事前零羨慕 (EFA) 和事前帕累托效率 (PEA)**。這意味著在設計機制時，需要權衡這些理想的屬性。例如，LIKE 機制對於 0/1 效用可以同時實現 EFA 和 PEA，但對於一般效用則無法同時實現。

## 4. 尋求漸進式保證

當完全的公平性難以保證時，我們可以尋求隨著時間推移或物品數量增加而趨於公平的機制。

• **羨慕消失 (Vanishing Envy)**：設計機制使最大羨慕值與物品數量之比，在物品數量趨於無限時趨近於零。研究表明，隨機分配物品可以在期望上實現羨慕消失，並且在漸進上接近最優。

## 5. 允許物品再分配 (Reallocating Items)

這是一種直接的技術，透過在新的物品資訊揭示後，調整過去的分配決定，來恢復或改善某些理想的規範屬性。

• 例如，對於兩個代理人、加性估值和線上抵達的物品，**即使是未獲取未來資訊的演算法，也需要重新分配 $\Theta(m)$ 個物品才能確保 EF1 (羨慕至多一件物品) 分配**。而對於三個或更多代理人，即使是獲取未來資訊的演算法也需要重新分配 $\Omega(m)$ 個物品來確保 EF1 分配。

## 6. 考慮計算複雜度與策略性行為

在處理線上分配時，理解代理人計算策略性行動的複雜性也很重要。

• 雖然某些線上機制（如 BALANCED LIKE）可能不具備策略穩健性，即代理人可能透過不誠實地報告偏好來獲益，但研究也表明，**計算這些策略性行動通常是難以處理的問題**。這意味著，即使存在理論上的策略空間，代理人也可能無法實際計算並執行最佳策略。這表明在策略穩健性和公平分配之間存在權衡。

總而言之，處理線上公平分配問題需要綜合運用這些方法，包括開發適應線上特性的新機制、放寬對規範屬性的要求、理解不可能性限制、追求漸進式公平以及在必要時允許再分配
