在中國大陸，由於防火牆（俗稱「長城防火牆」GFW）的存在，民眾若要訪問被封鎖的境外網站（如 Google、YouTube、Twitter 等），必須**「翻牆」**。翻牆方式五花八門，但主要的原理大多繞不開幾個核心：**流量加密、流量偽裝、出口轉發**。以下是最常見的幾種方式：

---

### 🔐 一、VPN（Virtual Private Network）

**最常見的翻牆方式之一。**

- 使用者與境外伺服器建立加密通道。
    
- 全部流量都經過 VPN 傳送。
    

#### 優點：

- 一鍵連接簡單方便。
    
- 可保護隱私、防止中間人攻擊。
    

#### 缺點：

- **容易被 GFW 掃描特徵封鎖**（除非有混淆或特殊協議）。
    
- 商業 VPN 容易被封 IP 或封端口。
    

---

### 🚀 二、代理（VPS + 代理協議）

中國大陸非常多技術使用者自己架設 VPS（Virtual Private Server），然後跑一些**代理協議**。

#### 常見協議：

1. **Shadowsocks（SS）**
    
    - 開源、輕量、支援加密。
        
    - 現已被 GFW 能力掃描到（尤其是傳統版）。
        
2. **ShadowsocksR（SSR）**
    
    - SS 的加強版，提供混淆功能。
        
    - 雖然被認為更難偵測，但已逐步被 GFW 辨識。
        
3. **V2Ray（VMess 協議）**
    
    - 更靈活、支援多種協議與混淆（如 TLS、WebSocket）。
        
    - 能偽裝成正常 HTTPS 流量，是目前最穩的主流之一。
        
4. **Trojan**
    
    - 設計目標是與真實的 HTTPS 幾乎無法區分。
        
    - 常配合 TLS + Nginx 使用。
        
    - 目前非常流行，GFW 難以精確封鎖。
        

---

### 🌐 三、DNS 解封、SmartDNS、DoH/DoT

這類技術主要是針對「**DNS 污染**」問題，而不是全流量翻牆。

- DNS 解封：解決域名解析被污染的問題，但本身**不能翻牆**。
    
- DoH（DNS over HTTPS）、DoT（DNS over TLS）：加密 DNS 查詢，防止被攔截。
    
- SmartDNS：將訪問某些網站的域名指向境外 IP，但可能會被封。
    

⚠️ **只能輔助翻牆，不能單獨達成完整的牆外通訊。**

---

### 🌀 四、Tor、I2P 等匿名網路

- **Tor**：透過分層加密路由，實現高度匿名。
    
- 目前在中國的 Tor 公開節點多數被封，需使用「橋接節點（bridge）」。
    

⚠️ 連線速度慢、易受干擾。

---

### 📱 五、加密隧道 / 雙重代理

- 有些人會將流量先導向境內中繼機器，再轉出到國外，進一步規避封鎖。
    
- 例如：
    
    - Shadowsocks → HTTP/2 over TLS → 國外出口
        
    - V2Ray + TLS + WebSocket over CDN
        

---

### 總結：在中國常見的翻牆方式比較

|方法|普及度|穩定性|抗封鎖性|難度|
|---|---|---|---|---|
|VPN|高|中|低～中|低|
|SS / SSR|中|中|低～中|中|
|V2Ray|高|高|高|中～高|
|Trojan|高|高|高|中～高|
|DNS 解封|中|低|低|低|
|Tor|低|低|中|高|

---

許多「翻牆黨」會採用下列高級技術混淆流量：

- TLS-in-TLS（雙層加密）
    
- CDN 前置（利用 Cloudflare 等作為跳板）
    
- 雙重代理（國內跳板 → 境外跳板）

## 🔥 1. CDN 前置（Cloudflare 前置）

### 💡 原理：

透過將流量「包裝」在 Cloudflare（或其他 CDN）提供的 HTTPS 流量中，使 GFW 難以封鎖具體 IP 或協議。這類流量對 GFW 來說幾乎 indistinguishable from normal traffic。

### 🧰 實作方式（以 V2Ray 為例）：

1. 架設一台境外 VPS（如日本、韓國、香港）
    
2. 綁定一個域名，並將它託管到 Cloudflare。
    
3. 設定 Cloudflare：
    
    - 開啟 Proxy（橘色雲）
        
    - 自動 HTTPS 重寫
        
    - 支援 WebSocket
        
4. 在 VPS 上部署 V2Ray：
    
    - 使用 **WebSocket + TLS**
        
    - 埠號使用 443（HTTPS 標準埠）
        
5. 配置 nginx 作為反向代理：
    
    ```nginx
    location /ray {
        proxy_redirect off;
        proxy_pass http://127.0.0.1:10000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
    ```
    
6. 客戶端配置 V2Ray：
    
    - 使用相同的域名（Cloudflare 解析）
        
    - VMess + WebSocket + TLS，路徑為 `/ray`
        

### 🚧 優點：

- 無法封鎖 IP（因為用的是 Cloudflare IP）
    
- 難以區分是否為「牆內目標」還是正常站點
    

### ⚠️ 缺點：

- Cloudflare 有風控，濫用可能導致被封域名或 403。
    
- 設計不當容易洩露特徵（如 V2Ray headers）。
    

---

## 🔐 2. TLS-in-TLS（雙層加密）

### 💡 原理：

把 TLS 流量包在另一層 TLS 中，達成「HTTPS 中的 HTTPS」，例如 Trojan 本身就使用 TLS，再透過 Nginx 再加 TLS。

### 🧰 實作方式：

1. VPS 上安裝 Nginx，配置 HTTPS（證書可用 Let's Encrypt）
    
2. Trojan 埠綁定到 localhost:4433
    
3. Nginx 將 `/trojan` 路徑反代到 127.0.0.1:4433
    
4. 客戶端連線：
    
    - `https://yourdomain.com/trojan`
        
    - 第一層 TLS 是 Nginx，第二層是 Trojan
        

### 🚧 優點：

- 讓 DPI 難以解析真正的應用協議
    
- 增加偽裝能力
    

### ⚠️ 缺點：

- 較高延遲
    
- 複雜度提升，配置錯誤會 GG
    

---

## 🔁 3. 雙重代理（跳板代理）

### 💡 原理：

牆內與牆外之間建立「中繼」，如：

- 用戶端 → 國內中繼（非翻牆）→ 境外出口（真正代理）  
    這樣可以繞過 GFW 對境外伺服器的 IP 偵測與封鎖。
    

### 🧰 實作方式：

1. 在牆內租一台國內主機（阿里雲、騰訊雲、青龍面板等）
    
2. 部署一個簡單代理程式（如 Hysteria、SOCKS、SSH Tunnel）
    
3. 牆外部署 V2Ray 或 Trojan 等
    
4. 本地用戶先連內部代理，再由其轉發到牆外主機
    

### 🚧 優點：

- 國內主機不易被 GFW 阻擋
    
- 可動態更換出口 IP，增加存活率
    

### ⚠️ 缺點：

- 成本較高（需租兩台主機）
    
- 國內代理若被舉報或監控，風險增加
    

---

## 🧊 4. V2Ray（VMess 協議）

### 💡 原理：

VMess 是 V2Ray 的原生協議，支援用戶驗證與流量混淆，能搭配 WebSocket、TLS、HTTP2 等傳輸層協議，具有高度偽裝能力。

### 🧰 實作方式（簡版）：

1. VPS 安裝 V2Ray
    
2. 使用 vmess 協議、WebSocket 傳輸
    
3. 可搭配 TLS 與 CDN 前置
    
4. 產生 JSON 配置檔，提供給用戶端
    

```json
{
  "inbounds": [{
    "port": 443,
    "protocol": "vmess",
    "settings": { ... },
    "streamSettings": {
      "network": "ws",
      "security": "tls",
      "wsSettings": { "path": "/ray" }
    }
  }]
}
```

### 🚧 優點：

- 靈活、插件化
    
- 支援多種偽裝與協議鏈接
    

### ⚠️ 缺點：

- 預設特徵已被 GFW 掌握，需搭配 TLS 或 CDN 使用
    

---

## 🦅 5. Trojan（Trojan-GFW）

### 💡 原理：

Trojan 是專門為了「完美偽裝成 HTTPS」設計的代理協議，使用者憑密碼通過 TLS 握手，否則就是一個正常網站（由 nginx 提供靜態頁）。

### 🧰 實作方式：

1. VPS 上安裝 Trojan
    
2. 使用合法 TLS 證書（Let's Encrypt）
    
3. 配置 nginx 提供靜態網頁作為偽裝站
    
4. 使用者輸入正確密碼時進入代理通道
    

### 🚧 優點：

- 非常難以區分是否為翻牆連線
    
- 真正達成「零特徵」
    

### ⚠️ 缺點：

- 配置需小心，錯了就變成靜態網站 🤡
    
- 依賴 TLS 證書，證書過期就不能用了

## 一、系統與環境準備

1. **作業系統**：Ubuntu 22.04 LTS（其他 Debian/Ubuntu 皆類似）
    
2. **root 權限**：或有 `sudo` 的使用者
    
3. **域名**：已在 Cloudflare 加入，且可切換「Proxy 狀態」（橘雲）

## 二、安裝 V2Ray

```bash
#!/usr/bin/env bash
set -e

# 1. 下載並安裝官方安裝腳本
bash <(curl -L https://github.com/v2fly/fhs-install-v2ray/raw/master/install-release.sh)

# 2. 產生 UUID（用於 VMess 認證）
UUID=$(cat /proc/sys/kernel/random/uuid)
echo "你的 VMess UUID：$UUID"

# 3. 建立基本 config，存於 /usr/local/etc/v2ray/config.json
cat > /usr/local/etc/v2ray/config.json <<EOF
{
  "inbounds": [{
    "port": 10000,
    "protocol": "vmess",
    "settings": {
      "clients": [
        {
          "id": "$UUID",
          "flow": "xtls-rprx-vision"
        }
      ]
    },
    "streamSettings": {
      "network": "ws",
      "wsSettings": {
        "path": "/ray"      // 後面可隨機化
      }
    }
  }],
  "outbounds": [{
    "protocol": "freedom",
    "settings": {}
  }]
}
EOF

# 4. 啟用並啟動
systemctl enable v2ray
systemctl restart v2ray
```

> **小彩蛋**：`flow: "xtls-rprx-vision"` 是 XTLS 的一種可選優化，能進一步隱藏流量特徵。

## 三、設定 Nginx 反向代理 + TLS

### 1. 安裝 Nginx & Certbot

```bash
sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx
```

### 2. Nginx 配置範例（`/etc/nginx/sites-available/v2ray.conf`）

```nginx
server {
    listen 443 ssl http2;
    server_name your.domain.com;

    # TLS 憑證（初次可用 Certbot 自動設定）
    ssl_certificate /etc/letsencrypt/live/your.domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your.domain.com/privkey.pem;
    ssl_session_cache shared:SSL:10m;
    ssl_protocols TLSv1.2 TLSv1.3;
    # 可依需求開啟更嚴格的安全參數...

    location /ray {
        # 隱藏真實後端，改成隨機路徑也可如 /$(openssl rand -hex 5)
        proxy_redirect off;
        proxy_pass http://127.0.0.1:10000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        # 混淆 UA 與其他 header，可選
        proxy_set_header User-Agent "Mozilla/5.0";
    }

    # 用於 HTTP → HTTPS 的跳轉
    error_page 497 https://$host$request_uri;
}

server {
    listen 80;
    server_name your.domain.com;
    # 強制跳轉到 HTTPS
    return 301 https://$host$request_uri;
}
```

啟用並重載：

```bash
ln -s /etc/nginx/sites-available/v2ray.conf /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

## 四、在 Cloudflare 做前置（橘雲）

1. **DNS**：將 `your.domain.com` A 記錄指向你的 VPS IP，Proxy 狀態切換到橘雲。
    
2. **SSL/TLS**：設定為「Full (strict)」模式。
    
3. **混淆策略**（可選）：
    
    - 啟用 **HTTP/2**、**TLS 1.3**
        
    - 關閉 **TLS 0.9/1.0/1.1**
        

如此一來，對方只看到 Cloudflare IP，流量又被包在 HTTPS + HTTP/2 裡，幾乎零特徵。

## 五、自動更新憑證（Certbot）

```bash
# 取得初次證書
sudo certbot --nginx -d your.domain.com --non-interactive --agree-tos -m your@email

# 查看自動任務
systemctl list-timers | grep certbot

# 測試更新流程
sudo certbot renew --dry-run
```

Certbot 預設會把新的憑證放到 `/etc/letsencrypt/live/your.domain.com/`，Nginx 自動重載。

## 六、隱藏路徑與防偵測策略

1. **隨機化 WebSocket 路徑**
    
    - 可用小 script 每天改一次 `/ray` 為 `/$(head /dev/urandom | tr -dc a-z0-9 | head -c6)`，並同步更新 V2Ray & Nginx。
        
2. **Header 混淆**
    
    - 在 Nginx 裡用 `proxy_set_header` 改變 `User-Agent`、`Referer`、`Accept-Encoding` 等，模仿正常網站。
        
3. **流量分流**
    
    - 結合 Clash / sing-box 等客戶端，僅對目標站點走 V2Ray，其他走直連。
        
4. **TLS 指紋偽裝**
    
    - V2Ray 可以用 XTLS 或者 SHADOWTLS 插件，改動 TLS ClientHello 參數，避免偵測。
