CLAHE (Contrast Limited Adaptive Histogram Equalization，限制對比度自適應直方圖均衡化)

傳統的直方圖均衡化 (HE) 雖然能提升影像對比度，但很容易把影像中的「雜訊」也跟著放大，導致畫面看起來充滿顆粒感或過度曝光。**CLAHE 就是為了解決這個問題而誕生的進階改良版。**

投影片將 CLAHE 的運作原理解構成四個主要步驟，並附上了一段實作程式碼：

### **CLAHE 的四大步驟解析**

1. **Partition images into tiles (將影像切割成多個小區塊)：** 不把整張圖片混在一起處理，而是將影像切成許多小網格（通常是 8x8，如底下程式碼所示）。
    
2. **Contrast Limiting (限制對比度與重新分配) —— 這是最核心的一步：**
    
    - 請看中間的圖表：圖表代表某個小區塊內的像素亮度分佈（直方圖）。如果某個亮度的像素「太多了」（形成一個很高的尖峰），直接做均衡化會產生強烈的雜訊。
        
    - 因此，系統會設定一個**門檻值 (predefined value)**（圖中的水平灰線）。
        
    - 超過這條線的山峰會被「削平」(clip)（圖中紫色的部分）。
        
    - 被削掉的像素並不是丟掉，而是被**平均分配 (distribute equally)** 到所有的亮度區間（墊高了底部的藍色區塊）。這樣既保留了細節，又限制了對比度過度放大。
        
3. **Apply HE for each tile (對每個小區塊進行直方圖均衡化)：** 在完成上述的「削峰填谷」後，才對這個小區塊執行標準的均衡化運算。
    
4. **Apply interpolation (應用內插法消除區塊邊界)：** 因為我們是把圖片切成一格一格處理的，如果不做後處理，圖片拼接起來會有明顯的「馬賽克」方塊邊界感。透過雙線性內插法 (Bilinear Interpolation)，可以平滑化這些邊界，讓整張圖片看起來自然連續。

### **底部的 OpenCV 程式碼在做什麼？**

投影片最下方的 Python 程式碼，示範了如何在實際專案中使用 CLAHE：

1. `cv2.createCLAHE(...)`：建立一個 CLAHE 物件，設定對比度限制為 `2.0`，網格大小為 `8x8`。
    
2. `cv2.cvtColor(image, cv2.COLOR_BGR2HSV)`：這是一個非常重要的實務技巧！它**沒有**直接處理彩色圖片的 RGB，而是先轉成 **HSV 色彩空間**。
    
3. `hsv_image[:,:,2] = clahe.apply(...)`：HSV 中的 **V (Value)** 代表亮度。它**只對亮度通道**套用 CLAHE。這樣做是為了提升明暗對比，但**不會扭曲原本的色彩 (Hue) 和飽和度 (Saturation)**。
    
4. 最後再把圖片轉回 BGR 格式以便顯示。
