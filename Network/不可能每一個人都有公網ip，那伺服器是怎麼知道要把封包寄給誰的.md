當你使用 NAT（網路位址轉換）時，其實並不是每一台電腦都需要直接擁有公網 IP。內部網路（私有 IP）中的電腦會通過 NAT 裝置（例如路由器）發出連線，NAT 裝置會在發出封包時將內部 IP 與 Port 轉換成自己的公網 IP 與一個對應的 Port，同時在 NAT 轉換表中建立一個對應關係。

當外部伺服器回覆時，封包會送到 NAT 裝置的公網 IP 及該 Port，NAT 裝置就會根據先前建立的對應關係，把封包正確地轉發到內部那台發起連線的電腦上。

因此，伺服器其實只「看到」發出請求的公網 IP 與 Port，而回覆封包時，NAT 裝置負責把回應送回正確的內部機器。這就是為什麼即使大部分人沒有直接的公網 IP，仍然能夠進行網路通信的原因。

這個過程可以參考 [Wikipedia 的 NAT 條目](https://zh.wikipedia.org/wiki/NAT) 進一步了解原理。
