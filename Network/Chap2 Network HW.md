### 1. 網站如何追蹤使用者？

**常見答案內容：**

- **Cookies 與 Session 機制：** 網站常用 cookies 記錄使用者的唯一識別碼，並配合伺服器端的 session 來追蹤狀態。
    
- **本地存儲技術：** 如 localStorage 或 sessionStorage，可儲存使用者資訊。
    
- **指紋辨識 (Fingerprinting)：** 利用瀏覽器特徵、裝置資訊等進行識別。
    
- **URL參數或隱藏表單欄位：** 部分應用也可能利用 URL token 來辨識使用者。
    

**答案解析與評論：**

- **優點：** 答案若能涵蓋以上主要方法，則能完整說明現代網站如何在不同層面（用戶端與伺服器端）實現追蹤。
    
- **可改進處：** 若解釋中能進一步討論各種技術的安全性與隱私疑慮（例如 cookie 被竊取的風險，指紋辨識帶來的隱私問題），會更完整。
    
- **補充說明：** 有些答案可能會提到第三方 cookies 與跨網站追蹤的情境，這也是值得注意的議題。

### 2. HTTP/1.1 的 HOL 阻塞問題與 HTTP/2 的解決方案

**常見答案內容：**

- **HTTP/1.1 的問題：**
    
    - **HOL (Head-of-Line) 阻塞：** 在 HTTP/1.1 中，即使使用持久連線（Keep-Alive）與管道化（Pipelining），若前面的請求延遲回應，其後的請求也會被迫等待，造成整體效能降低。
        
- **HTTP/2 的改進：**
    
    - **多路復用 (Multiplexing)：** 在同一個連線上可以同時發送多個請求與接收回應，互不影響。
        
    - **二進位分幀：** 將傳輸內容拆分成多個獨立的幀 (frames)，從而解決了 HOL 阻塞問題。
        
    - **優先權控制：** 讓客戶端能夠為不同請求設定不同的優先順序。
        

**答案解析與評論：**

- **優點：** 若答案能明確指出 HTTP/1.1 中的排隊問題與 HTTP/2 透過多路復用、二進位分幀來消除這種限制，說明就很到位。
    
- **可改進處：** 建議在說明中補充一些實際的效能提升數據，或舉例說明當前場景下如何改善延遲問題。
    
- **補充說明：** 可以再強調一下，在 HTTP/2 中，不僅僅是解決了 HOL 阻塞，同時也使得連線利用率更高，這對現代網頁的快速加載十分重要。

### 3. 遞歸與迭代 DNS 查詢的差異

**常見答案內容：**

- **遞歸查詢 (Recursive Query)：**
    
    - 使用者向 DNS 解析器發出查詢，解析器負責向其他 DNS 伺服器查詢並返回最終結果。
        
- **迭代查詢 (Iterative Query)：**
    
    - 使用者（或本地解析器）向 DNS 伺服器查詢後，若該伺服器無法直接給出答案，會返回下一個可查詢的 DNS 伺服器地址，由使用者自行發起後續查詢。
        

**答案解析與評論：**

- **優點：** 答案若能清楚區分「由解析器主動遞歸」與「由客戶端主動依序查詢」的運作模式，就能夠正確描述兩者差異。
    
- **注意點：** 注意題目中出現「DNA queries」可能為筆誤，實際上應為「DNS queries」，答題時應考慮這點並加以糾正。
    
- **可改進處：** 可以補充說明這兩種查詢方式在效能、安全性以及網路負載方面的影響，讓答案更全面。

### 4. BitTorrent 中只下載不上傳能否獲得完整檔案？

**常見答案內容：**

- **BitTorrent 協議機制：**
    
    - 協議強調互惠原則（tit-for-tat），用戶需要上傳一定數據才能獲得較高的下載速度。
        
- **Bob 的情境：**
    
    - 若 Bob 完全不上傳數據，他可能會被其他節點「限速」或「拒絕服務」，使得下載過程極度受阻。
        
    - 雖然在某些情況下，如果 swarm 中有足夠的種子 (seeders)，Bob 理論上可能最終下載到完整檔案，但這種情況較少見且可能會受到懲罰。
        

**答案解析與評論：**

- **優點：** 答案若能解釋 BitTorrent 如何利用交換機制來提升網路整體效能，並說明不貢獻上傳會影響下載速度甚至導致被其他用戶拒絕，則能清楚回應題意。
    
- **可改進處：** 可以進一步探討協議中存在的策略（例如上傳比率限制），以及現實中各大 tracker 如何處理純粹的「吸血」行為。
    
- **補充說明：** 強調協作精神是 P2P 網絡成功的關鍵，單方面的下載行為最終會破壞 swarm 的健康運作，這也是大部分 BitTorrent 客戶端會對無上傳者採取限制措施的原因。
